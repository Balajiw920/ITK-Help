Detailed Description

When objects are being modified on the server, it is important that the information be passed on the client(s) for the same modifications to be replicated there.
The Service Oriented Architecture (SOA) layer takes care of passing the information to the clients. It must, however, be first captured through the ITKs in this group.

Two lists are managed: the changed objects and the deleted objects.

TCCORE_API int TCTYPE_add_to_changed_tag_list ( tag_t  object,
const char *  properties 
)

Adds the specified object (and potentially its properties) to the list of objects that have changed.
If the parameter properties is a null pointer, all the loaded properties of the object are treated as changed.

Returns
ITK_ok always
Parameters
object(I) The tag of the object to be added to the list of changed objects.
properties(I) A string separated list of modified properties for this object.

TCCORE_API int TCTYPE_get_modified_tag_list ( logical  clear_lists,
int *  num_objects,
tag_t **  tags,
int **  deleted_or_modified,
char ***  properties 
)

Retrieves the objects that belong to the changed list and those that belong to the deleted list.

Returns
ITK_ok always
Parameters
clear_lists(I) Indicates if the lists will be reset after this call.
Valid values are false (do not reset the lists) and true.
num_objects(O) The total number of changed and deleted objects.
tags(OF) num_objects The list of tags for the objects.
deleted_or_modified(OF) num_objects Indicates what kind of update that has occurred on the object: 0 = deleted, 1 = changed.
properties(OF) num_objects Array of property lists (string with property names separated by a comma) indicating the update that has occurred on the object.
An empty string denotes an unidentified change (possibly all the properties)
A value "name_property1,name_property2" denotes changes done to the properties "name_property1" and "name_property2".
The array is packed, which means that the memory is deallocated by simply calling MEM_free on the container (not its elements).
